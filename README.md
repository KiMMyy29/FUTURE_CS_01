# FUTURE_CS_01
# Website Security Assessment Report

Overview
This project presents a security assessment conducted on the Wikipedia website using ethical and non-intrusive techniques.

The objective was to identify common security weaknesses, assess risk levels, and provide practical recommendations in a professional audit format.

Scope
- Public-facing pages only
- Passive analysis only
- No exploitation or harmful actions performed

---

Tools Used
- Nmap – Port and exposure analysis
- OWASP ZAP (Passive Scan) – Vulnerability identification
- Browser Developer Tools – Header and configuration analysis

Key Findings
- Strong overall security posture
- Minimal exposed services (Ports 80 & 443 only)
- Secure HTTPS implementation
- Minor information disclosure via HTTP headers

Risk Classification
- Low Risk vulnerabilities identified
- No critical or high-risk issues found

Recommendations
- Reduce server information disclosure
- Continue enforcing strong HTTPS policies
- Maintain regular security monitoring and updates

Report
The full vulnerability assessment report is available in this repository.

Disclaimer
This assessment was conducted strictly under ethical guidelines. No intrusive techniques or exploitation methods were used.
